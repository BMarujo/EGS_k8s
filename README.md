# FlashSale Kubernetes Deployment

This directory deploys the full FlashSale microservice stack into the
`tenant-grupo2-egs-deti-ua-pt` namespace:

- Composer API gateway and React frontend
- Auth API and static auth frontend
- Payment Auth API
- Inventory API
- Payment API and wallet/checkout UI
- Postgres and Redis for each data-owning service
- OpenTelemetry Collector, Prometheus, Grafana, Jaeger, and dev Vault
- Traefik `Ingress` routes for the `*.flashsale` browser hostnames

## Deploy

```bash
kubectl apply -k k8s
kubectl rollout status deploy/composer
kubectl get pods,pvc,ingress
```

The app images are configured in `k8s/kustomization.yaml`. Rebuild and push them
with `k8s/scripts/build-and-push.sh` after code changes.

## Local DNS

The frontend and auth/payment browser pages use hostnames such as
`composer.flashsale`, `auth.flashsale`, and `payment.flashsale`. Map them to the
cluster edge IP on the machine where you open the browser:

```bash
sudo k8s/scripts/update-hosts.sh 193.136.82.35
```

Then open:

- Main app: http://composer.flashsale
- Composer API docs: http://composer.flashsale/docs
- KPI dashboard in the app: http://composer.flashsale, then sign in with a
  promoter account (`@prom.pt`) and open the KPI/dashboard view
- KPI JSON: `GET http://composer.flashsale/api/kpi/dashboard` with that
  promoter account's `Authorization: Bearer ...` token
- Prometheus metrics generated by Composer:
  `GET http://composer.flashsale/metrics`
- Grafana platform KPI dashboard:
  http://grafana.flashsale/d/flashsale-platform-kpis/flashsale-platform-kpis
- Grafana: http://grafana.flashsale (`admin` / `admin`)
- Prometheus: http://prometheus.flashsale
- Jaeger: http://jaeger.flashsale
- Payment wallet UI: http://payment.flashsale/wallet/login

For the checkout flow, the final payment authorization uses the separate
Payment Auth service. In the browser this happens via the wallet UI on
`payment.flashsale`, which calls `payment-auth.flashsale`.

## Smoke Test

```bash
k8s/scripts/smoke-test.sh 193.136.82.35
```

The smoke test creates a promoter, creates/publishes an event, provisions a
payment customer, authorizes a checkout, verifies `composer.flashsale/metrics`,
checks Prometheus for `flashsale_payment_payments_total`, and verifies the
provisioned Grafana dashboard.

Useful direct checks:

```bash
curl http://composer.flashsale/metrics | grep flashsale_
curl --get http://prometheus.flashsale/api/v1/query \
  --data-urlencode 'query=flashsale_service_up'
curl -u admin:admin \
  http://grafana.flashsale/api/dashboards/uid/flashsale-platform-kpis
```
